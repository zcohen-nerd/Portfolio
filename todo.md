# Portfolio Content Accuracy To-Do List

This checklist focuses on **credibility, accuracy, and trust** (not technical/site errors).

## 1) Lock your professional narrative (highest priority)
- [ ] Pick one primary identity and use it everywhere.
- [ ] Confirm current roles/titles and institution names exactly as they should appear publicly.
- [ ] Add a short “current scope” statement (what you do now, what was past work, what is personal R&D).
- [ ] Add date context to the homepage bio so readers can tell what is current vs historical.

## 2) Remove or qualify unverified claims
- [ ] For every numeric metric, either replace with a verified value + source note or convert to qualitative wording.
- [ ] Add a private evidence log mapping each public claim to internal proof.
- [ ] If a metric is estimated, explicitly label it as an internal estimate and add date range.

## 3) Separate project maturity honestly
- [ ] Classify each project as: **Concept**, **Prototype**, **Pilot**, **Production**, or **Deployed Personal System**.
- [ ] Ensure “status” labels match body text and claims.
- [ ] For student/teaching demos, avoid production-style language unless there was real production use.

## 4) Tighten page-level integrity
- [ ] Add context: where/when, role, team size.
- [ ] Add constraints: safety, budget, schedule, environment.
- [ ] Add direct contribution statements.
- [ ] Keep outcomes limited to defensible statements.

## 5) Fix inflation patterns in copy
- [ ] Remove unsupported absolutes.
- [ ] Avoid mixed confidence language.
- [ ] Keep scale claims consistent across README, homepage, and project pages.

## 6) Clarify teaching outcomes
- [ ] Tie outcomes to named course, term range, and sample size when published.
- [ ] Add context on measurement method (rubric, survey, exam, external data).
- [ ] Distinguish institution-level outcomes from outcomes you directly influenced.

## 7) Add transparency policy
- [ ] Add a short “Claims & Evidence” note explaining values may be rounded or redacted.
- [ ] Add “last updated” timestamps on key pages.
- [ ] Add a “selected work” disclaimer.

## 8) Quick wins
- [ ] Replace placeholder images/text with real artifacts or remove them.
- [ ] Remove any claim you cannot defend today.
- [ ] Add timeframe tags to major claims.
- [ ] Add one sentence under each project: “This page was last content-validated on YYYY-MM-DD.”
